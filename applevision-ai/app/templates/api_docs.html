API Documentation

Integrate AppleVision AI classification into your applications with our REST API.

Open API — No Authentication Required

All endpoints are currently publicly accessible.

Base URL: /api JSON Response

Endpoints

POST /api/predict GET /api/history GET /api/stats GET /api/health
POST /api/predict

Submit an apple image for classification. Returns the predicted variety, confidence score, top-5 predictions, and inference time.

Parameters

Name Type Required Description
file File (multipart/form-data) Required Apple image file (JPG, PNG, WEBP)

cURL Example

curl -X POST http://localhost:5001/api/predict \ -F "file=@apple_image.jpg"

Response — 200 OK

{ "top_class": "Granny Smith", "confidence": 0.9847, "inference_time_ms": 142.35, "is_high_confidence": true, "threshold": 0.75, "top_predictions": [ { "class_name": "Granny Smith", "score": 0.9847 }, { "class_name": "Golden 3", "score": 0.0089 }, { "class_name": "Golden 1", "score": 0.0032 }, { "class_name": "Braeburn", "score": 0.0018 }, { "class_name": "Pink Lady", "score": 0.0009 } ] }

Error Responses

400 No file provided or invalid file format
500 Model not loaded or prediction failed
GET /api/history

Retrieve the recent prediction history. Returns a list of past classification results with timestamps.

cURL Example

curl http://localhost:5001/api/history

Response — 200 OK

{ "history": [ { "id": 1, "top_class": "Granny Smith", "confidence": 0.9847, "timestamp": "2025-01-15T10:30:00Z" } ], "total": 42 }
GET /api/stats

Get aggregated statistics including total predictions, variety distribution, daily trends, and average confidence.

cURL Example

curl http://localhost:5001/api/stats

Response — 200 OK

{ "total_predictions": 1247, "most_predicted": "Granny Smith", "today_count": 23, "avg_confidence": 92.4, "variety_counts": { "Braeburn": 145, "Granny Smith": 203, // ... other varieties }, "daily_trend": [ { "date": "2025-01-15", "count": 23 } ] }
GET /api/health

Health check endpoint. Returns server status and model availability.

cURL Example

curl http://localhost:5001/api/health

Response — 200 OK

{ "status": "healthy", "model_loaded": true, "labels_loaded": true, "version": "2.0.0" }

Rate Limiting

Currently, there are no rate limits enforced. For production deployments, we recommend implementing rate limiting at the reverse proxy level (e.g., Nginx) to prevent abuse. A recommended limit is 60 requests per minute per IP for the prediction endpoint.

Need help integrating? Check out our GitHub repository for examples and more details.
